# To-Do List API (Flask + SQLite)

REST API для управления списком задач. Приложение написано на Python с использованием фреймворка Flask и ORM SQLAlchemy.
Реализован полный цикл CRUD-операций с сохранением данных в реляционную базу данных SQLite.

## Стек технологий

- **Язык:** Python 3
- **Web-фреймворк:** Flask
- **ORM (Работа с БД):** Flask-SQLAlchemy
- **База данных:** SQLite (хранение в файле `tasks.db`)
- **Формат обмена данными:** JSON

## Функциональные возможности

1.  **Персистентность данных:** Задачи сохраняются в файл базы данных и не исчезают после перезагрузки сервера.
2.  **Валидация данных:** Входящие JSON-запросы проверяются на наличие обязательных полей, типы данных и пустые строки. Сервер возвращает корректные ошибки (400 Bad Request) с описанием проблемы.
3.  **Гибкое обновление:** Поддерживается частичное обновление полей задачи (механизм PATCH через метод PUT).

## Описание API (Эндпоинты)

### 1. Получить все задачи
*   **URL:** `/tasks`
*   **Метод:** `GET`
*   **Ответ (200 OK):** Возвращает массив объектов задач.

### 2. Создать задачу
*   **URL:** `/tasks`
*   **Метод:** `POST`
*   **Тело запроса:**
    ```json
    {
      "title": "Название (обязательно)",
      "description": "Описание (опционально)"
    }
    ```
*   **Ответ (201 Created):** Возвращает созданный объект с присвоенным ID.

### 3. Получить задачу по ID
*   **URL:** `/tasks/<id>`
*   **Метод:** `GET`
*   **Ответ:** Объект задачи или `404 Not Found`.

### 4. Обновить задачу
*   **URL:** `/tasks/<id>`
*   **Метод:** `PUT` (поддерживает логику PATCH)
*   **Тело запроса:** Можно передавать только те поля, которые нужно изменить.
    ```json
    {
      "status": "done"
    }
    ```
*   **Ответ (200 OK):** Возвращает обновленный объект.

### 5. Удалить задачу
*   **URL:** `/tasks/<id>`
*   **Метод:** `DELETE`
*   **Ответ (200 OK):** `{"result": true}`
